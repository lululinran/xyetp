# 快速部署检查清单

## 🚀 5分钟快速部署（本地开发）

### 步骤1：环境检查（2分钟）

```bash
# 检查JDK
java -version  # 应该显示 1.8.x

# 检查Maven
mvn -version   # 应该显示 3.5+

# 检查MySQL
mysql --version  # 应该显示 5.7+ 或 8.0+
```

### 步骤2：数据库设置（1分钟）

```bash
# 登录MySQL
mysql -u root -p

# 执行以下SQL
CREATE DATABASE IF NOT EXISTS market CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
exit;

# 导入数据
mysql -u root -p market < src/main/resources/market.sql
```

### 步骤3：修改配置（1分钟）

编辑 `src/main/resources/jdbc.properties`：
```properties
username=root
password=你的MySQL密码  # 修改这里
```

### 步骤4：IDEA配置（1分钟）

1. 打开IDEA，导入项目
2. 右键 `src/main/resources` → Mark Directory as → Resources Root
3. File → Project Structure → Modules → 确认Sources和Resources正确标记

### 步骤5：运行项目

**方法A：IDEA直接运行**
1. Run → Edit Configurations
2. 添加 Tomcat Server (Local)
3. Deployment → 添加 `kd-second-hand-workshop:war exploded`
4. Application context: `/`
5. 端口：8088
6. 运行

**方法B：Maven打包后部署**
```bash
mvn clean package
# 将 target/kd-second-hand-workshop.war 复制到Tomcat的webapps目录
```

### 步骤6：访问测试

- 前台：http://localhost:8088/goods/index
- 后台：http://localhost:8088/admin/toLogin
- 测试账号：15232103749 / 123456

## ⚠️ 常见问题快速解决

### 问题1：启动后404
**解决**：
- 检查端口是否为8088
- 确认访问路径：`/goods/index`
- 查看Tomcat日志

### 问题2：数据库连接失败
**解决**：
- 检查MySQL是否启动：`systemctl status mysql`
- 检查jdbc.properties中的用户名密码
- 检查数据库market是否存在

### 问题3：图片上传失败
**解决**：
- 检查webapp目录权限
- 确认上传路径存在

### 问题4：中文乱码
**解决**：
- 确认数据库字符集：`SHOW CREATE DATABASE market;`
- 应该显示 `utf8mb4`
